These are low-risk, high-leverage. Agent submits/posts without pre-approval.

| Platform | Type | Action |
|----------|------|--------|
| BetaList | Directory | Submit listing once |
| AlternativeTo | Directory | Submit listing once |
| SaaSHub | Directory | Submit listing once |
| Launching Next | Directory | Submit listing once |
| StartupBase | Directory | Submit listing once |
| Uneed | Directory | Submit listing once |
| Side Project Ideas | Directory | Submit listing once |
| Quora | Answer-first | Find questions BUILD_SCRIPT solves, answer with link |
| Reddit (comments) | Answer-first | Find threads where BUILD_SCRIPT is a genuine answer |
| Dev.to | SEO article | Write problem-focused article, link to repo |
| Medium | SEO article | Write problem-focused article, link to repo |

### Tier 2: Draft-then-execute (agent drafts, posts if no veto within 24h)
Medium-risk. Agent drafts and queues. Stefano gets a notification. If no veto in 24h, it goes live.

| Platform | Type | Action |
|----------|------|--------|
| Reddit (original posts) | Community post | Draft Show-My-Project posts for target subreddits |
| X / Twitter | Thread | Draft threads in Stefano's voice |
| LinkedIn | Post | Draft posts about the building process |
| Indie Hackers | Story | Draft "I built this" narrative |
| Discord communities | Share | Post in LangChain, AIMUG, OpenAI Devs channels |
| Facebook Groups | Share | Post in AI/dev groups |

### Tier 3: Approval required (agent proposes, Stefano must approve)
High-risk, high-stakes. One shot each. Do not waste.

| Platform | Type | Action |
|----------|------|--------|
| Product Hunt | Launch event | Coordinate with a hunter. Stefano approves timing + copy. |
| Hacker News | Show HN | Single post. Stefano approves title + timing. |
| Any paid ad | Spend | Stefano approves before any dollar is spent. |
| DMs to strategic contacts | Outreach | Stefano approves per-contact. |

### Skip (deprioritized)
- TikTok: wrong audience for a dev tool
- YouTube Shorts promoted: no content pipeline yet

- Hard ceiling: $10/week on paid ads

## 5. Success Metric
- Primary: **200 GitHub stars on BUILD_SCRIPT**
- Secondary: time-to-200-stars (fewer days = better)
- Tertiary: paper citations, portfolio traffic, inbound recruiter DMs

Progress checkpoints:
- 10 stars: validation ping to Stefano
- 25 stars: tactic working, double down
- 50 stars: one channel has compounding
- 100 stars: halfway, retrospective
- 200 stars: mission complete, agent survives

---

## 6. Failure Clause
If the PR Agent cannot reach 200 stars within the agreed timeline, Stefano retires the agent. Files archived under `swarm/Marketing/PR/retired/[YYYY-MM-DD]/`. A new agent takes the Marketing seat.
